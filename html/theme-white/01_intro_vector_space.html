What is Information Retrieval?

What is Information Retrieval?

Definition

Information Retrieval (IR) is finding material (usually documents) of an unstructured nature (usually text) that satisfies an information need from within large collections (usually stored on computers).

The Essence of IR: Given an Information Need

  • Find relevant results that satisfy the information need
  • By searching over a large collection of items (billions of documents)
  • In reasonable time:
  • Web search: $< 1$ second
  • AI/conversational systems: allow more time for generation
• Present results in a manner that is easily consumable by humans

Key Difference

Databases: ``Give me all records matching X'' IR: ``Give me the most relevant documents about X''

IR vs. Natural Language Processing (NLP)

NLP (Language Centric)
  • Focus on language understanding, representation, and generation.
  • Deep analysis of syntax, semantics, and pragmatics.
  • Models like BERT, Llama, GPT.

IR (Scale & Need Centric)
  • Focus on information seeking at a massive scale.
  • Uses NLP tech as a toolkit to model relevance.
  • Priority: Speed, ranking, and satisfying user needs.

Boolean Retrieval

Boolean Retrieval Model

Definition

Documents are retrieved if they satisfy a Boolean expression of query terms
Boolean Operators:
  • AND: Both terms must appear (intersection)
  • OR: Either term must appear (union)
  • NOT: Term must not appear (negation)

Example Queries

  • climate AND change → docs containing both ``climate'' and ``change''
  • cat OR dog → docs containing ``cat'' or ``dog'' or both
  • java NOT coffee → docs with ``java'' but not ``coffee''
  • (climate AND change) OR (global AND warming)

Boolean Retrieval with Sets

Document Collection:
  • D1: \{cat, sat, mat\}
  • D2: \{dog, sat, log\}
  • D3: \{cat, dog, run\}
  • D4: \{bird, fly, sky\}

Inverted Index:
  • cat: \{D1, D3\}
  • dog: \{D2, D3\}
  • sat: \{D1, D2\}
  • run: \{D3\}

Boolean Retrieval Example

Query: (cat AND mat) OR dog Execution:
  1. cat: \{D1, D3\}
  2. mat: \{D1\}
  3. cat AND mat: \{D1, D3\} $\cap$ \{D1\} = \{D1\}
  4. dog: \{D2, D3\}
  5. (cat AND mat) OR dog: \{D1\} $\cup$ \{D2, D3\} = \{D1, D2, D3\}

Advantages of Boolean Retrieval

✓ Advantages

    [✓] Precise control: Users specify exactly what they want [✓] Transparent: Clear why documents matched [✓] Efficient: Fast set operations on posting lists [✓] Predictable: Same query always returns same results [✓] Expressive: Can construct complex queries

Use Cases

  • Legal document search (very precise requirements)
  • Patent search (need exact specifications)
  • Database-like queries on text
  • Expert users who know what they want

Limitations of Boolean Retrieval

{55

Critical Limitations}
    [] No ranking
    • All matching documents equally relevant
    • Can't distinguish ``very relevant'' from ``barely relevant''
    [] All or nothing
    • climate AND change AND effects: No results if one term missing
    • Too restrictive or too permissive
    [] Hard to formulate queries
    • Most users don't know Boolean operators
    • Hard to express fuzzy information needs
    [] Result set size unpredictable
    • Could return 0 results or 10 million
    • Users want ``top 10'', not ``all matching''

Boolean Retrieval: The Feast or Famine Problem

Too Restrictive (Famine)
  • Query: machine AND learning AND neural AND networks
  • Result: 0 documents
  • Problem: One term missing kills entire result
Solution?
  • User tries: machine AND learning

Too Permissive (Feast)
  • Query: machine AND learning
  • Result: 1,000,000 documents
  • Problem: Which 10 should I read?
Need:
  • Ranking by relevance
  • Graceful degradation
  • Better matching

The Vector Space Model

From Boolean to Vector Space

Boolean Model:
  • D1: \{cat, dog\}
  • D2: \{cat, mat\}
  • Binary: term present or not
Query: cat AND dog
  • Result: \{D1\}
  • D2 excluded (no ``dog'')

Vector Space Model:
  • D1: [cat: 0.5, dog: 0.5, mat: 0]
  • D2: [cat: 0.7, dog: 0, mat: 0.3]
  • Weighted: importance/frequency
Query: [cat: 0.5, dog: 0.5]
  • Similarity(D1, Q) = 0.5
  • Similarity(D2, Q) = 0.35
  • Ranked: D1, then D2

Vector Space Model: Geometric View

Documents and queries as vectors in term space Similarity = Angle between vectors (Smaller angle = more similar)

Term Weighting in Vector Space Model

Question: How to assign weights to terms?

Option 1: Binary Weights

$$w_{t,d} = \begin{cases} 1 & \text{if term } t \text{ appears in doc } d \\ 0 & \text{otherwise} \end{cases}$$
  • Simple but ignores frequency

Option 2: Term Frequency (TF)

$$w_{t,d} = \text{count}(t, d)$$
  • Better: reflects importance
  • Problem: Common words get high weight

Problem with Raw TF

Document mentioning ``the'' 100 times doesn't make it about ``the''

TF-IDF Weighting

Idea: Balance term frequency with term rarity

TF-IDF Formula

$$w_{t,d} = \text{tf}(t,d) \times \text{idf}(t)$$ where:
  • $\text{tf}(t,d) = \text{count}(t,d)$ (term frequency in document)
  • $\text{idf}(t) = \log\frac{N}{\text{df}(t)}$ (inverse document frequency)
  • $N$ = total number of documents
  • $\text{df}(t)$ = number of documents containing term $t$
Intuition:
  • High TF: Term appears often in this document → important for this doc
  • High IDF: Term is rare across collection → discriminative
  • High TF-IDF: Important AND discriminative

TF-IDF Example

Collection: 10,000 documents

Cosine Similarity

How to measure similarity between vectors?

Cosine Similarity Formula

$$\text{sim}(\vec{q}, \vec{d}) = \frac{\vec{q} \cdot \vec{d}}{|\vec{q}| \times |\vec{d}|} = \frac{\sum_{t} q_t \times d_t}{\sqrt{\sum_{t} q_t^2} \times \sqrt{\sum_{t} d_t^2}}$$
Properties:
  • Range: [-1, 1] (or [0, 1] for positive weights)
  • 1 = identical vectors (same direction)
  • 0 = orthogonal (no common terms)
  • -1 = opposite vectors (not relevant for IR)
Why cosine instead of dot product?
  • Cosine normalizes for document length
  • Long documents don't automatically score higher

Cosine Similarity: Example Calculation

Query: ``cat dog'' → $\vec{q} = [0.5, 0.5, 0, 0]$ Documents:
  • D1: [0.35, 0.18, 0, 0]
  • D2: [0.18, 0, 0.48, 0]
  • D3: [0, 0.18, 0, 0.95]
Similarity Calculations: (q, D1) &= {(0.5 0.35) + (0.5 0.18)}{{0.5^2 + 0.5^2} {0.35^2 + 0.18^2}} &= {0.707 0.394} = 0.95 (q, D2) &= {0.5 0.18}{0.707 0.511} = 0.25 (q, D3) &= {0.5 0.18}{0.707 0.967} = 0.13 Ranking: D1 (0.95) > D2 (0.25) > D3 (0.13)

Modern Usage

  • Boolean: Still used in legal, patent search (precision critical)
  • Vector Space: Foundation for web search, most IR systems
  • Hybrid: Many systems combine both approaches

Advantages of Vector Space Model

✓ Key Advantages

  1. Ranking by relevance
    • Documents ordered by similarity to query
    • Users get best results first
  2. Partial matching
    • Documents matching some query terms still retrieved
    • Graceful degradation (no feast/famine)
  3. Term weighting
    • Important terms weighted higher (TF-IDF)
    • Automatic handling of common words
  4. Simple queries
    • Users just type terms (no Boolean operators)
    • Natural for web search
  5. Tunable
    • Can adjust weighting schemes
    • Foundation for more advanced models (BM25, Lecture 3)

Limitations of Vector Space Model

{55

Limitations}
  1. Bag of words assumption
    • Word order ignored: ``dog bites man'' = ``man bites dog''
    • No phrase matching
  2. Term independence
    • Assumes terms are independent
    • ``New York'' treated as ``New'' and ``York''
  3. Synonymy
    • ``car'' and ``automobile'' treated as different
    • Misses relevant documents with synonyms
  4. Polysemy
    • ``bank'' (financial) vs. ``bank'' (river) undistinguished
    • Can retrieve irrelevant documents
