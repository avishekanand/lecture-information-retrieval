What is Question Answering?

From extracting spans to open-domain generation

The Information Need

Evolution of Search

  • 1990s Web Search: ranked documents
  • 2011 IBM Watson: direct answer
  • 2018 BERT: extract answer span
  • 2020 GPT-3: generate answer from weights
  • 2020 RAG: retrieve then generate

The Core Tension

  • Retrieval gives us evidence but cannot reason.
  • Generation gives us fluency but can fabricate.
  • QA needs both.

What Does a QA System Do?

Input: natural language question. Output: direct answer.

Document IR: returns a ranked list of passages.

QA: returns a single concise answer.

Why harder? Questions are short/ambiguous. Evidence may be split. Answer must be correct, not just relevant.

Two Ways to Know

Parametric (LLMs)

  • Knowledge baked into weights
  • Fast inference, no index
  • Fails: stale data, rare facts, confident errors

Non-parametric (Retrieval)

  • External, updatable index
  • Explicit, citable, fresh
  • Fails: retrieval errors propagate; cannot reason alone

RAG = Parametric + Non-Parametric

This lecture's focus: Open-domain QA. RAG is the dominant approach.

Open-Domain QA: The Setup

  • Input: Question $q$ and Corpus $\mathcal{C} = \{d_1, d_2, \dots, d_N\}$
  • Step 1: Retrieve top-$k$ passages $\mathcal{R}_k(q) \subset \mathcal{C}$
  • Step 2: Generate answer $a = f(q, \mathcal{R}_k(q))$

If the evidence isn't retrieved, probability of correct answer approaches 0.
All QA quality is bounded by retrieval quality.

Evaluation Limits

Task-Level Metrics

  • Exact Match / Token F1: Good for spans, bad for abstraction. automobile ≠ car
  • BERTScore/ROUGE: Better for generated abstract text.
  • Question: Is the answer correct?

System-Level Metrics

  • Faithfulness
  • Attribution
  • Hallucination Rate
  • Question: Is every claim grounded in evidence?

A system can score high on EM by memorizing training answers while hallucinating on novel queries.

From Reader to Generator

Retriever-Reader Paradigm

DrQA (2017) established the 'Retrieve then Read' framework.
  • Stage 1: TF-IDF Retriever maximizes passage recall.
  • Stage 2: BiLSTM Reader maximizes span precision.

Retriever-Reader-Generator

Closed-Book QA: The LLM Bet

LLMs answering exclusively from their parametric weight memory.

Why Closed-Book QA Falls Short

Failures

  • Knowledge cutoff: no access to post-training facts
  • Long-tail failure: rare entities poorly represented
  • No evidence: cannot cite sources
  • Hallucination: confident generation of false content

The Scale Paradox

Larger models hallucinate more confidently, not less.

Scale alone does not solve factual grounding.

The Hallucination Crisis

Factually incorrect content presented with confidence

What Is Hallucination?

  • Intrinsic: contradicts the given context or source
  • Extrinsic: asserts facts absent from any source (pure fabrication)

Why it's not a bug

  • LLMs maximize token likelihood, not truth
  • No external factuality check at inference
  • Fluent generation ≠ faithful generation

Hallucinations in the Wild

High-Stakes Hallucination

Why LLMs Hallucinate

Root Causes

  • Training data noise (contradictions)
  • Distributional gaps (unseen entity combos)
  • Autoregressive bias (no global fact-check)
  • RLHF prioritizes helpfulness over truth

The Core Problem

  • Generates the most likely continuation
  • Does not generate the most true continuation

RAG as the Solution

Grounding

Grounding generation in retrieved evidence constrains the model.

  • Retrieved evidence is explicit and checkable.
  • Prompts force citations: "use only passages"
  • Index updateable without retraining

Key Insight

  • Errors shift from invisible hallucinations to detectable retrieval failures.
  • Retrieval defines reasoning substrate.

RAG 101

Pipeline of Decisions

What RAG Actually Is

A compound AI system, not a single model trick.

  • Retrieval defines reasoning substrate.
  • If evidence is not retrieved, it cannot be used.
  • Modularity allows isolated debugging.

Pipeline: Retrieval $\to$ Conditioning $\to$ Generation

A System View

Treat each module as a component with knobs and an evaluation contract.

Sequential Training in RAG

Options

  1. Stage 1: Pre-train/fine-tune dense retriever
  2. Stage 2: Joint fine-tuning (marginalise over retrieved passages)
  3. Stage 3: RLHF/DPO for grounding preferences

Practical default (2025)

  • Freeze the retriever (Contriever/e5-large)
  • LoRA-fine-tune generator on grounded QA
  • Joint training is expensive and often unnecessary

Lewis et al. 2020

The paper that formalized RAG

Key Innovation

Retrieval treated as a latent variable.

$$p(y \mid x) \approx \sum_{z} p(z \mid x)\, p(y \mid x, z)$$

  • Marginalizing over documents makes retrieval differentiable.
  • RAG-Sequence: retrieve once for full answer.
  • RAG-Token: retrieve per generated token.

The Ceiling

Most "hallucinations" in RAG are selection failures: the evidence existed but was not retrieved.

The Evidence Ceiling

Logic

$$\text{Answer}(q) = f(q, \mathcal{R}_k(q))$$

  • Retrieve: find the right chunk
  • Select: include it in context window
  • Use: force grounding via prompt

Implication

  • Improving the generator yields zero gain if retrieval recall is the bottleneck.
  • Measure Recall@k before tuning generator.

Chunking: A Modeling Decision

The retrieval unit determines recall, precision, and context efficiency.
  • Fixed-size: simple; may split sentences
  • Semantic: boundary at paragraph; variable length
  • Recursive: balances size and coherence
  • Extraction: tables, PDFs

Rule: A chunk without provenance metadata breaks the attribution chain.

Late Chunking

Context-Aware Embeddings (Günther et al., 2024)

Encode the full document first, then mean-pool chunk representations.
Massive coherence boost with same storage footprint.

Query Reformulation

Strategies

  • HyDE: Generate hypothetical document, embed that.
  • Multi-Query: Generate paraphrases, retrieve all, fuse (RRF).
  • Step-back: abstract to a broader concept
  • Decomposition: Break down multi-part questions.

Danger: Intent Drift

Reformulation can shift user intent.

Always monitor retrieval overlap between original and rewritten queries to detect drift away from actual need.

Retrieval Quality Metrics

  • Recall@k: does the gold evidence appear in top-k?
  • MRR: mean reciprocal rank of first relevant passage
  • Context Hit Rate: fraction of queries where retrieved context contains sufficient information to answer correctly (measured against ground truth).

Measurement order: Measure Recall@k first. If < 0.7, fix the retriever before tuning the generator.

Beyond Single-Shot

Multi-Step Retrieval

The DeRIVeR Paradigm

Single-shot limitation

$q \to \mathcal{R}_k(q) \to a$

One retrieval round captures only the surface question. Fails on multi-hop/ambiguous queries.

DeRIVeR

  • 1. Query Planner
  • 2. Retrieval Module
  • 3. Reasoning Synthesis

Each role can be independently optimized, swapped, or scaled.

Query Planning

Example

Q: "CEO of company that acquired DeepMind's education?"

  • $q_1$: "Company acquired DeepMind" $\to$ Google
  • $q_2$: "CEO of Google" $\to$ Sundar Pichai
  • $q_3$: "Education Sundar Pichai" $\to$ IIT

Strategies

  • Sequential: $q_i$ uses $q_{i-1}$ result
  • Parallel: independent, then fuse
  • Conditional: branch on evidence

Cost Trade-off: Each step adds latency/tokens. Measure first.

Single-Shot vs. Multi-Step Retrieval

Iterate until confidence is met or budget exhausted.
  • Single-shot: retrieve once; fast but fails on ambiguity
  • Multi-step: retrieve $\to$ reason $\to$ retrieve again

Evidence Chaining: Multi-Hop Reasoning

Latent links: bridging entities not in original query.

Failure mode: If the bridge entity is not retrieved correctly, the entire chain collapses. Error propagates.

Search-in-the-Loop

Retrieval as an LLM tool-call within reasoning trace.
  • Trigger: search when model uncertainty $u_t > \theta$
  • Budget: cap max tool calls
  • Training: RLHF to learn when not to search

Rule of Thumb

Only add complexity when a clear recall gap exists and latency/cost budget permits.
