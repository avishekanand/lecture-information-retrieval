Today's Question

We can store and retrieve efficiently. But how do we score?

The Probability Ranking Principle (PRP)

Theorem (Robertson, 1977)

If documents are ranked by decreasing probability of relevance given the query, then the expected effectiveness of the system is maximized---under the assumption of independent relevance.
Formally: Rank by $P(\text{relevant} \mid q, d)$ Implications:
  • Gives a theoretical justification for probabilistic scoring
  • All scoring functions (TF-IDF, BM25, LM) are approximations of this ideal
  • Independence assumption is strong---violated in practice (documents are correlated)

Key Insight

The PRP tells us to optimize. BM25 and Language Models tell us .

The Ranking Problem

Formal Definition

Given:
  • Query $q$ (sequence of terms)
  • Document collection $D$
  • Inverted index (from Lecture 2)
Return: Ranked list scored by relevance $P(\text{relevant}|q,d)$

Term Weighting: TF-IDF

Term Frequency (TF)
  • More occurrences $\rightarrow$ more important
  • $\text{tf}(t,d) = \text{count}(t,d)$
Inverse Document Frequency (IDF)
  • Rare terms $\rightarrow$ more discriminative
  • $\text{idf}(t) = \log\frac{N}{\text{df}_t}$

Combined Score

$$\text{TF-IDF}(t,d) = \text{tf}(t,d) \times \text{idf}(t)$$

BM25 Formula

Complete Formula

$$\text{BM25}(q,d) = \sum_{t \in q} \text{IDF}(t) \cdot \frac{f(t,d) \cdot (k_1+1)}{f(t,d) + k_1 \cdot \left(1-b+b \cdot \frac{|d|}{\text{avgdl}}\right)}$$
Three components:
  1. IDF: Term importance (like TF-IDF)
  2. Saturating TF: Non-linear term frequency
  3. Length normalization: Penalize long documents
Standard parameters: $k_1 = 1.2$, $b = 0.75$

BM25 Component 1: IDF

IDF Formula

$$\text{IDF}(t) = \log\frac{N - \text{df}_t + 0.5}{\text{df}_t + 0.5}$$
where:
  • $N$ = total number of documents
  • $\text{df}_t$ = document frequency of term $t$
Properties:
  • Similar to TF-IDF but probabilistically motivated
  • Can be negative for very common terms ($\text{df} > N/2$)
  • Rare terms get high positive IDF
  • Common terms get low or negative IDF

BM25 Component 2: Saturating TF

Saturating TF Formula

$$\frac{f(t,d) \cdot (k_1+1)}{f(t,d) + k_1}$$
Parameter $k_1$ (typically 1.2--2.0):
  • $k_1 \to 0$: Boolean matching (TF doesn't matter)
  • $k_1 \to \infty$: Linear TF (like TF-IDF)
  • $k_1 = 1.2$: Moderate saturation (standard)

BM25 Component 3: Length Normalization

Length Normalization Term

$$1 - b + b \cdot \frac{|d|}{\text{avgdl}}$$
Parameter $b$ (typically 0.75):
  • $b = 0$: No length normalization
  • $b = 1$: Full normalization (penalize long docs heavily)
  • $b = 0.75$: Partial normalization (standard)
Why needed:
  • Long documents have more terms just by chance
  • Should we penalize or not? Depends on collection
  • $b$ controls the trade-off

The Generative Story

Intuition

Imagine randomly sampling words from a document. The document that is most likely to produce your query terms is ranked highest.
How it works step by step:
  1. Pick a document $d$ from the collection
  2. The document defines a language model --- a probability distribution $P(w|d)$ over words
  3. Sample query words from this distribution: $P(q|d) = \prod_{w \in q} P(w|d)$
  4. Rank documents by $P(q|d)$ --- those more likely to ``emit'' the query rank higher

Query Likelihood Model: Philosophy

The Generative Story

Instead of matching keywords, we ask: ``What is the probability that this document would produce this query?''
Mathematical Formulation:
  • Assume query terms $w \in q$ are sampled independently from the document $d$ (unigram assumption).
  • Rank documents by: $$P(q|d) = \prod_{w \in q} P(w|d)$$
  • In practice, we use log-probabilities to avoid underflow: $$\text{score}(q,d) = \sum_{w \in q} \log P(w|d)$$

Estimation

Maximum Likelihood Estimate (MLE)

How do we calculate $P(w|d)$? The simplest way is term frequency: $$P_{\text{ML}}(w|d) = \frac{\text{count}(w,d)}{|d|}$$
But what is the catch here?

The Catch

If a query term $w$ is not in the document, then $P_{\text{ML}}(w|d) = 0$.
  • The product $P(q|d) = 0$ for the entire query!
  • One missing term kills the score, even for a 90% relevant document.
Example: Query ``machine learning algorithms'' misses a document about ``machine learning'' because the word ``algorithms'' is absent.

Smoothing: Core Idea

Principle

Mix document evidence with collection evidence: $$P(w|d) = f(\text{document evidence}, \text{collection evidence})$$
Collection Language Model: $$P(w|C) = \frac{\sum_{d' \in C} \text{count}(w,d')}{\sum_{d' \in C} |d'|}$$

Effect

Even if $\text{count}(w,d) = 0$, we have $P(w|d) > 0$ from collection

Jelinek-Mercer Smoothing

Formula

$$P(w|d) = (1-\lambda) \cdot P_{\text{ML}}(w|d) + \lambda \cdot P(w|C)$$
Parameter $\lambda$ (typically 0.1--0.3):
  • $\lambda = 0$: Only use document (no smoothing)
  • $\lambda = 1$: Only use collection (ignore document)
  • $\lambda = 0.1$: Mostly trust document, small collection backup
Properties:
  • ✓ Simple and interpretable
  • ✓ Same $\lambda$ for all documents
  • Doesn't adapt to document length

Dirichlet Smoothing

Formula

$$P(w|d) = \frac{\text{count}(w,d) + \mu \cdot P(w|C)}{|d| + \mu}$$
Parameter $\mu$ (typically 1000--2500):
  • Smoothing strength (number of ``pseudo-counts'' from collection)
  • Higher $\mu$: More smoothing. Lower $\mu$: Less smoothing.
Key Difference: Length-Adaptive
  • Short documents: $\mu$ dominates denominator $\rightarrow$ rely more on collection
  • Long documents: Actual counts dominate $\rightarrow$ rely on document
  • Automatically adapts smoothing to document length!

Query Likelihood Scoring: Step by Step

The Recipe

Given query $q$ and document $d$, score = $\log P(q|d)$:
  1. Estimate via MLE: $\displaystyle P_{\text{ML}}(w|d) = \frac{\text{count}(w,d)}{|d|}$
  2. Smooth (JM): $\hat{P}(w|d) = (1-\lambda)\,P_{\text{ML}}(w|d) + \lambda\,P(w|C)$
  3. Score: $\displaystyle\text{score}(q,d) = \sum_{w \in q} \log \hat{P}(w|d)$
  4. Rank by score --- higher score = more relevant.

Relevance Feedback Overview

Explicit Relevance Feedback

User marks documents as relevant/non-relevant
  • Impractical: users won't give feedback
  • Chicken-and-egg: need good initial results

Pseudo Relevance Feedback (PRF)

Assumption: Top-$k$ retrieved documents are relevant
  • ✓ No user interaction needed
  • ✓ Automatic query expansion
  • Risk: If assumption fails $\rightarrow$ query drift
Process:
  1. Retrieve initial results with original query
  2. Extract terms from top-$k$ documents
  3. Expand query with these terms
  4. Re-retrieve with expanded query

RM3: Relevance Model 3

Provides a probabilistic way to perform query expansion by building a model of relevance ($R$) from the initial top-$k$ results. Phase 1: Estimating the Relevance Model $P(w|R)$ We want to find terms $w$ that are likely to appear in relevant documents: $$P(w|R) \approx \sum_{d \in \text{top-}k} \underbrace{P(w|d)}_{\text{term presence}} \cdot \underbrace{P(d|q)}_{\text{doc relevance weight}}$$
  • $P(w|d)$ is the smoothed language model of document $d$.
  • $P(d|q)$ is the score of the document from the first retrieval stage.
  • Terms common across many high-scoring documents get the highest $P(w|R)$.

RM3: Interpolation and Final Query

Phase 2: Combining with the Original Query To prevent the query from drifting too far, we interpolate the relevance model with the original query's word

distribution:

$$P(w|q_{\text{new}}) = (1-\lambda) P(w|q_{\text{orig}}) + \lambda P(w|R)$$
  • $\lambda=1$: Full query expansion (total trust in top-$k$).
  • $\lambda=0$: No expansion (original query only).
  • Common default: $\lambda \approx 0.5$.

The Final Retrieval

The top fb_terms from $P(w|q_{\text{new}})$ are used to form a new weighted query for the second stage.

RM3: Key Parameters

fb_docs (Feedback Documents): 3--10

How many top documents to use for expansion
  • Too few: May miss relevant terms. Too many: Risk non-relevant docs.
  • Standard: 10

fb_terms (Feedback Terms): 10--20

How many expansion terms to add
  • Too few: Limited benefit. Too many: Noise.
  • Standard: 10

$\lambda$ (Interpolation Weight): 0.5--0.8

Original vs. expansion weight
  • Lower (0.5): Trust original query more
  • Higher (0.8): Trust expansion more

RM3 Example

Original Query: ``machine learning'' Top-3 Retrieved Documents (fb_docs=3):
  1. ``Neural networks and deep learning algorithms...''
  2. ``Supervised machine learning techniques...''
  3. ``Machine learning applications in AI...''
Extracted Terms (fb_terms=5):
  • neural (P=0.15), deep (P=0.12), supervised (P=0.10), algorithms (P=0.09), AI (P=0.08)
Expanded Query ($\lambda=0.6$): ``machine learning neural deep supervised algorithms AI''

Key Insight

Strategy 3: Chain-of-Thought Query Decomposition --- Intuition

The Problem

``How did the 2008 financial crisis affect European housing markets?'' No single document covers all aspects. BM25 on the full query retrieves almost nothing.

Core Idea

Decompose complex queries into atomic sub-queries, retrieve evidence for each independently, then merge the results. More recall, less vocabulary mismatch.

Strategy 4: GenQR (Generative Query Reformulation) --- Intuition

The Problem with ``term lists'' expansions

Synonyms/HyDE/Query2Doc can help, but often produce {bag-of-terms} or long pseudo-text. Sometimes you just want a better query that expresses the same intent.

Core Idea (Wang et al., 2022)

Use an LLM to rewrite the query itself into 3--5 high-quality paraphrases, then retrieve using the reformulated query (often appended/interpolated with the original).

Key point vs PRF

No dependence on top-$k$ pseudo-relevant docs: GenQR is direct reformulation using only $q_0$.

What We Covered Today

Sparse, Term-Based Ranking

  • BM25: Probabilistic relevance, saturating TF, length normalization
  • Language Models: Generative probabilities, smoothing essential
  • Query Expansion: RM3 for vocabulary mismatch + LLM-based alternatives

Fundamental Limitations

  • Still bag-of-words (order doesn't matter)
  • Exact term matching only (even with expansion)
  • No semantic understanding
  • ``car'' and ``automobile'' completely different
  • ``bank'' (river) vs. ``bank'' (financial) treated identically
